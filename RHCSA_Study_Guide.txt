Understand and use essential tools
------------------------------------
complete | Access a shell prompt and issue commands with correct syntax | Solution: Through GUI its in top left Applications > Utilities > Terminal
complete | Use input-output redirection (>, >>, |, 2>, etc.) | Solution: > sends output to file, >> appends output to file, | pipes output to a command, 2> outputs standard error to a file.
complete | Use grep and regular expressions to analyze text | Solution: man grep
complete | Access remote systems using ssh | Solution: man ssh
complete | Log in and switch users in multiuser targets | Solution: ssh user@host; su user2switch
complete | Archive, compress, unpack, and uncompress files using tar, star, gzip, and bzip2 | Solution: Create gzip: tar -czvf nameoftar.tar /directories /to /tar, Create bzip2: tar -cjvf nameoftar.tar /directories /to /tar
complete | Create and edit text files | Solution: Vi is a text editor installed on every linux system
complete | Create, delete, copy, and move files and directories | Solution: Delete files: rm /file/to/remove, Move files: mv /file/to/move /new/location, Copy files: cp /file/to/copy /new/copy
complete | Create hard and soft links | Solution: ln -s /target/to/create/link link_name, -s is for soft links
complete | List, set, and change standard ugo/rwx permissions | Solution: man chmod
complete | Locate, read, and use system documentation including man, info, and files in /usr/share/doc | Solution: use man for quick documentation. Other import files are generaly found in /usr/share/doc
Note: Red Hat may use applications during the exam that are not included in Red Hat Enterprise Linux for the purpose of evaluating candidate's abilities to meet this objective.

Operate running systems
-------------------------
complete | Boot, reboot, and shut down a system normally | Solution: systemctl reboot
complete | Boot systems into different targets manually | Solution: systemctl isolate
complete | Interrupt the boot process in order to gain access to a system | Solution: append rd.break to kernel parameters
complete | Identify CPU/memory intensive processes, adjust process priority with renice, and kill processes | Solution: man nice, renice
complete | Locate and interpret system log files and journals | Solution: /var/log 
complete | Access a virtual machine's console | Solution: virt-manager
complete | Start and stop virtual machines | Solution: virsh start domain
complete | Start, stop, and check the status of network services | Solution: systemctl
complete | Securely transfer files between systems | Solution: scp

Configure local storage
-------------------------
complete | List, create, delete partitions on MBR and GPT disks | Solution: man parted
complete | Create and remove physical volumes, assign physical volumes to volume groups, and create and delete logical volumes | Solution: man pvcreate, vgcreate, lvcreate
complete | Configure systems to mount file systems at boot by Universally Unique ID (UUID) or label | Solution: extract UUID from the blkid command
complete | Add new partitions and logical volumes, and swap to a system non-destructively | Solution: man parted, man lvcreate

complete | Extend existing logical volumes | Solution: man lvextend
complete | Create and configure set-GID directories for collaboration | Solution: man chmod
complete | Create and manage Access Control Lists (ACLs) | Solution: man getacl, setacl
complete | Diagnose and correct file permission problems | Solution: man chmod

Deploy, configure, and maintain systems
---------------------------------------
complete | Configure networking and hostname resolution statically or dynamically | Solution: modify /etc/hosts, /etc/resolv.conf/ etc/nsswitch.conf, /etc/hostname
complete | Schedule tasks using at and cron | Solution: configure system cron in /etc/crontab, local cron with command crontab -e
complete | Start and stop services and configure services to start automatically at boot | Solution: systemctl enable/disable servicename
complete | Configure systems to boot into a specific target automatically | Solution: systemctl set-default
complete | Install Red Hat Enterprise Linux automatically using Kickstart | Solution:  research kickstart solutions.
complete | Configure a physical machine to host virtual guests | Solution: install following packages: qemu-kvm, libvirt, libvirt-client, virt-install, virt-manager, virt-top, virt-viewer
complete | Install Red Hat Enterprise Linux systems as virtual guests | Solution: virt-install
complete | Configure systems to launch virtual machines at boot | Solution: virsh autostart domain_name
complete | Configure network services to start automatically at boot | Solution: systemctl enable service
complete | Configure a system to use time services | Solution: configure /etc/ntp.conf
complete | Install and update software packages from Red Hat Network, a remote repository, or from the local file system | Solution: man yum, yum.conf
complete | Update the kernel package appropriately to ensure a bootable system | Solution: rpm -ivh kernel_version.rpm *TAKE SNAPSHOT FIRST*
